TinnyBackend Router

A lightweight, type-safe HTTP routing system for Node.js with built-in middleware, cookie management, static file serving, and admin monitoring.

Attach custom metadata to server

Architecture Overview

The server is built around a handler registry pattern. Each route is registered with a method, path, handler function, optional middleware array, and an optional next callback. When a request arrives, the server iterates through registered handlers, matches by method and path, executes middleware chain, then the main handler.

Key Feature: The res.isClosed flag prevents multiple responses to the same request, eliminating a common class of bugs in HTTP servers.
Quick Start
import Server from './server.js'; const app = new Server({ port: 3000, hostname: 'localhost' }); app.add({ method: 'GET', path: '/hello', handler: (req, res) => res.send(200, { message: 'Hello World!' }) }); app.listen(); // Server running at http://localhost:3000

Core Methods

.listen(enableMon?, callback?, login?)

Starts the HTTP server on the configured port.

  • enableMon (boolean): If true, launches admin monitor on port + 1
  • callback (function): Called when server starts listening
  • login (LoginOpt): { username, password } for monitor authentication
app.listen(true, () => { console.log('Server + monitor running'); }, { username: 'admin', password: 'secure123' });
.decorate(name, data) & .useDecorator(name)

Store and retrieve custom data on the server instance. Useful for sharing database connections, configs, or service instances across handlers.

server.decorate('db', databaseConnection); server.decorate('config', { version: '2.0' }); // Later in a handler: const db = server.useDecorator('db'); const config = server.useDecorator('config');
.getHost() & .getPort()

Returns the configured hostname and port number respectively.

console.log(server.getHost()); // "localhost" console.log(server.getPort()); // 3000

Session Management (Internal)

The server internally tracks sessions with IP-based identification, request counting, and ban intervals for rate limiting:

// Internal session structure { ip: string, // Client IP address lastReq: number, // Timestamp of last request reqCount: number, // Number of requests key: string, // Random session key (32 hex chars) banInterval: number // 60000ms (1 minute) }

Path Normalization

All paths automatically have trailing slashes removed (except root /). For example, /api/users/ becomes /api/users. This prevents route mismatches.

Middleware

Execute functions before the main handler for authentication, logging, validation, etc.

How Middleware Works

Middleware functions are executed in order before the main handler. Each middleware receives (req, res). If a middleware sends a response (e.g., 401), the isClosed flag prevents the main handler from executing.

Authentication Middleware
const requireAuth = (req, res) => { const token = res.getCookie('auth_token'); if (!token || token.value !== 'valid-secret') { res.send(401, { error: 'Unauthorized access' }); return; // isClosed = true, main handler won't run } }; app.add({ method: 'GET', path: '/admin/dashboard', handler: adminDashboardHandler, middelWares: [requireAuth] });
Request Logger Middleware
const requestLogger = (req, res) => { const start = Date.now(); const originalSend = res.send; res.send = function(...args) { const duration = Date.now() - start; console.log(`[${new Date().toISOString()}] ${req.method} ${req.url} - ${duration}ms`); return originalSend.apply(this, args); }; }; app.add({ method: 'GET', path: '/api/data', handler: dataHandler, middelWares: [requestLogger] });
Input Validation Middleware
const validateBody = (req, res) => { if (!req.body || !req.body.email) { res.send(400, { error: 'Email is required' }); return; } if (!req.body.email.includes('@')) { res.send(422, { error: 'Invalid email format' }); return; } }; app.add({ method: 'POST', path: '/subscribe', handler: subscribeHandler, middelWares: [validateBody] });

The next Callback

After the main handler executes (and if the response isn't closed), the optional next function runs. This is useful for cleanup, logging, or post-processing:

app.add({ method: 'DELETE', path: '/cache', handler: clearCacheHandler, next: (req, res) => { console.log('Cache cleared successfully'); // Perform additional cleanup... } });

Static Files & Directories

Serve entire directories automatically with MIME type detection.

servDir() Method

server.servDir(directoryPath, urlPrefix)

Recursively reads a directory and creates GET routes for every file. Files named index.* are also mapped to the directory path as default files.

ParameterDescription
directoryPathRelative path to the directory (e.g., "./public")
urlPrefixURL prefix for the routes (e.g., "assets")
// Serve ./public directory at /assets/* await server.servDir('./public', 'assets'); // This creates routes like: // GET /assets/style.css → ./public/style.css // GET /assets/js/app.js → ./public/js/app.js // GET /assets → ./public/index.html (auto-index)

SendFile Method

server.SendFile(res, filePath)

Internal method used by servDir(). Can also be called directly to serve individual files:

app.add({ method: 'GET', path: '/download/report', handler: (req, res) => { server.SendFile(res, 'reports/summary.pdf'); } });

404 Handling for Files

If a requested file doesn't exist, the server attempts to serve public/404.html. If that also fails, it returns a 500 error.

Admin Monitor

Built-in web dashboard for server administration on a separate port.

Enabling the Monitor

.listen(true, callback, {username, password})

When enableMon is true, a second server instance starts on port + 1 with the following routes:

RouteMethodAuthDescription
/GETNoLogin page or admin dashboard (if authenticated)
/adminGETYesMain admin dashboard
/api/loginPOSTNoAuthenticate with username/password
/admin/statusGETYesServer status endpoint
/css/style.cssGETNoLogin page styles
/docs/*GETNoServes ./public/doc directory
/imgs/*GETNoServes ./public/imgs directory

Authentication Flow

  1. User visits / → sees login form
  2. Submits credentials to POST /api/login
  3. On success, server sets cookie adminkey=yassine
  4. Subsequent requests check this cookie via isAuthed()
  5. Protected routes use Auth middleware
// Start server with monitor app.listen(true, () => { console.log('Main server: http://localhost:3000'); console.log('Monitor: http://localhost:3001'); }, { username: 'admin', password: 'mySecretPass' });
Security Note: The default cookie value is hardcoded as "yassine". For production use, implement proper session tokens and HTTPS.

Complete Example

import Server from './server.js'; const app = new Server({ port: 3000, hostname: '0.0.0.0' }); // Decorate with shared resources app.decorate('startTime', Date.now()); // Authentication middleware const auth = (req, res) => { const session = res.getCookie('session_id'); if (!session) { res.send(401, { error: 'Please login first' }); } }; // Public route app.add({ method: 'GET', path: '/', handler: (req, res) => res.send(200, { status: 'running', uptime: Date.now() - app.useDecorator('startTime') }) }); // Protected route with middleware app.add({ method: 'GET', path: '/api/secret', handler: (req, res) => res.send(200, { data: 'top-secret' }), middelWares: [auth], next: (req, res) => console.log('Secret accessed') }); // POST with body parsing app.add({ method: 'POST', path: '/api/echo', handler: (req, res) => res.send(200, { received: req.body, cookies: res.getAllCookies() }) }); // Serve static assets await app.servDir('./public', 'static'); // Start with admin monitor app.listen(true, () => { console.log('App: http://0.0.0.0:3000'); console.log('Admin: http://0.0.0.0:3001'); }, { username: 'admin', password: 'secure123' });

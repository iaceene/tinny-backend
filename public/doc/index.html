🚀 Routing System

Lightweight TypeScript Backend API Documentation

Overview

A tiny but powerful HTTP server built with Node.js and TypeScript featuring:

  • Dynamic routing with GET, POST, PUT, DELETE methods
  • Cookie-based authentication system
  • Middleware support
  • Automatic directory serving with index.html defaults
  • Built-in file serving with MIME type detection
  • Session management

API Endpoints

GET /
Root endpoint - shows login page or admin page based on authentication
GET /admin AUTH REQUIRED
Admin dashboard page (redirects to login if not authenticated)
POST /api/login
Authenticate user and receive admin cookie
Request:
{
  "username": "admin",
  "password": "password"
}
Success Response (200):
{
  "name": "admin",
  "password": "password"
}
Error Response (403):
{
  "Error": "Access Denied"
}
GET /admin/status AUTH REQUIRED
Get server status and file information
Response (200):
{
  "ok": "fw"
}
GET /css/style.css
Serve login page CSS stylesheet
GET /docs/*
Serve documentation files from public/doc directory

Authentication

The system uses cookie-based authentication with the key adminkey.

Login Flow:

  1. POST credentials to /api/login
  2. Server validates and sets adminkey cookie
  3. Use the cookie to access protected routes
  4. Protected routes verify cookie before serving content

Quick Start

1. Build the project:
tsc
2. Start the server:
node ./dist/index.js
3. Access the application:
http://localhost:3000
4. Login with credentials:
curl -X POST http://localhost:3000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"password"}'

File Serving

Automatically serve HTML, CSS, JS, images and JSON files from the public/ directory.

Supported MIME Types:

  • .html → text/html
  • .css → text/css
  • .js → text/javascript
  • .png → image/png
  • .jpg → image/jpeg
  • .json → application/json

Note: index.html files are automatically served as directory defaults

Features

Middleware Support

Routes support middleware functions that execute before the main handler, useful for authentication and validation.

Session Management

Track user sessions by IP address with automatic key generation and ban intervals.

Cookie Management

Built-in cookie manipulation with getter, setter, and cookie persistence across requests.

Directory Auto-Serving

Use servDir() to automatically create routes for all files in a directory with index file support.

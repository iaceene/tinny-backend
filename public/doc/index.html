TinnyBackend Framework

A lightweight, type-safe HTTP server with routing, middleware, cookies, static files, and built-in admin monitoring.
HTTP Methods

GET · POST · PUT · DELETE

Cookie Engine

Native get/set/add with parsing

Middleware Chain

Pre-handler execution with next()

Static Directory

Auto-index with MIME detection

Admin Monitor

Built-in dashboard with JWT auth

Decorators

Attach custom metadata to server

Key Features

  • TypeScript-first with full type definitions
  • Path parameters with automatic parsing (/users/:id)
  • Middleware support with chain execution
  • Cookie management with get/set/add operations
  • Static file serving with MIME type detection
  • Admin monitor with JWT authentication
  • Decorators for sharing data across routes
  • Built-in logging with colored console output
  • Session management with IP-based tracking
Quick Start: Create a server instance, add routes, and start listening in just a few lines of code.

Getting Started

Install and set up your first TinnyBackend server.

Installation

bash 
npm install tinny-backend

Basic Example

typescript 
import Server from 'tinny-backend';

const app = new Server({ port: 3000, hostname: 'localhost' });

app.add({
  method: 'GET',
  path: '/hello',
  handler: (req, res) => {
    res.send(200, { message: 'Hello World!' });
  }
});

app.listen(true, () => {
  console.log('Server running at http://localhost:3000');
});

Environment Variables

Create a .env file for admin monitor authentication:

env 
ADMIN_USERNAME=admin
ADMIN_PASSWORD=secure_password_here
ADMIN_KEY=your_secret_jwt_key
Important: Never commit your .env file to version control. Always use strong passwords and JWT secrets.

Server Class

The core class that creates and manages the HTTP server instance.

Constructor

OptionTypeDefaultDescription

ServerNamestring"Server"Name used in logs

Core Methods

.listen(enableMon?, callback?)

Starts the HTTP server on the configured port.

  • enableMon (boolean): If true, launches admin monitor on port + 1
  • callback (function): Called when server starts listening
example 
app.listen(true, () => {
  console.log('Server + monitor running');
});
.decorate(name, data) & .useDecorator(name)

Store and retrieve custom data on the server instance. Useful for sharing database connections, configs, or service instances across handlers.

example 
server.decorate('db', databaseConnection);
server.decorate('config', { version: '2.0' });

// Later in a handler:
const db = server.useDecorator('db');
const config = server.useDecorator('config');
.getHost() & .getPort()

Returns the configured hostname and port number respectively.

example 
console.log(server.getHost()); // "localhost"
console.log(server.getPort()); // 3000
.log(message, type?)

Log messages with timestamp and color coding. Messages are also stored in the internal logs array.

example 
server.log('User authenticated', 'message');
server.log('Database connection failed', 'error');

Routing

Register routes with method, path, handlers, and middleware.

Adding Routes

.add(option: AddOption)
PropertyTypeRequiredDescription
methodMethodsYes"GET" | "POST" | "PUT" | "DELETE"
pathstringYesURL path (supports :params)
handlerHandlerFunYesMain request handler function
middelWaresHandlerFun[]NoArray of middleware functions
nextHandlerFunNoCallback executed after main handler
example 
server.add({
  method: 'POST',
  path: '/api/users/:id',
  handler: (req, res) => {
    const userId = req.params.id;
    res.send(200, { user: userId, data: req.body });
  },
  middelWares: [authMiddleware, loggerMiddleware],
  next: (req, res) => {
    console.log('Response sent for:', req.url);
  }
});

Path Parameters

Define dynamic routes using :param syntax. Parameters are automatically parsed and available in req.params.

example 
app.add({
  method: 'GET',
  path: '/users/:userId/posts/:postId',
  handler: (req, res) => {
    const { userId, postId } = req.params;
    res.send(200, { userId, postId });
  }
});

Middleware

Execute functions before the main handler for authentication, logging, validation, etc.

How Middleware Works

Middleware functions are executed in order before the main handler. Each middleware receives (req, res). If a middleware sends a response (e.g., 401), the isClosed flag prevents the main handler from executing.

Authentication Middleware
example 
const requireAuth = (req, res) => {
  const token = res.getCookie('auth_token');
  if (!token || token.value !== 'valid-secret') {
    res.send(401, { error: 'Unauthorized access' });
    return; // isClosed = true, main handler won't run
  }
};

app.add({
  method: 'GET',
  path: '/admin/dashboard',
  handler: adminDashboardHandler,
  middelWares: [requireAuth]
});
Request Logger Middleware
example 
const requestLogger = (req, res) => {
  const start = Date.now();
  const originalSend = res.send;
  
  res.send = function(...args) {
    const duration = Date.now() - start;
    console.log(`[${new Date().toISOString()}] ${req.method} ${req.url} - ${duration}ms`);
    return originalSend.apply(this, args);
  };
};

The next Callback

After the main handler executes (and if the response isn't closed), the optional next function runs. This is useful for cleanup, logging, or post-processing:

example 
app.add({
  method: 'DELETE',
  path: '/cache',
  handler: clearCacheHandler,
  next: (req, res) => {
    console.log('Cache cleared successfully');
  }
});

Static Files

Serve entire directories automatically with MIME type detection.

.servDir() Method

.servDir(directoryPath, urlPrefix, middleware?)

Recursively reads a directory and creates GET routes for every file. Files named index.* are also mapped to the directory path as default files.

ParameterDescription
directoryPathRelative path to the directory (e.g., "./public")
urlPrefixURL prefix for the routes (e.g., "assets")
middlewareOptional middleware array for all routes
example 
// Serve ./public directory at /assets/*
await server.servDir('./public', 'assets');

// With authentication middleware
await server.servDir('./private', 'admin', [requireAuth]);

.SendFile() Method

.SendFile(res, filePath, status)

Internal method used by servDir(). Can also be called directly to serve individual files. Automatically falls back to 404.html if file not found.

example 
app.add({
  method: 'GET',
  path: '/download/report',
  handler: async (req, res) => {
    await server.SendFile(res, 'reports/summary.pdf', 200);
  }
});

Cookies

Complete cookie management with get, set, and add operations.

Cookie Methods

MethodDescription
res.getCookie(name)Get cookie by name, returns Cookie | null
res.setCookie(name, value)Update existing cookie value
res.addCookie(name, value)Add new cookie (skips if exists)
res.getAllCookies()Returns all cookies as string[]
example 
app.add({
  method: 'POST',
  path: '/login',
  handler: (req, res) => {
    // Set a cookie
    res.addCookie('session_id', 'abc123');
    
    // Get a cookie
    const session = res.getCookie('session_id');
    
    // Update a cookie
    res.setCookie('session_id', 'xyz789');
    
    // Get all cookies
    const allCookies = res.getAllCookies();
    
    res.send(200, { 
      message: 'Cookie set',
      cookies: allCookies 
    });
  }
});
Note: Cookies are automatically parsed from the Cookie header and stored in memory for the duration of the request.

Admin Monitor

Built-in web dashboard for server administration with JWT authentication.

Enabling the Monitor

.listen(true, callback)

When enableMon is true, a second server instance starts on port + 1 with the following features:

Authentication Flow

  1. User visits / → sees login form
  2. Submits credentials to POST /api/login
  3. On success, server sets JWT cookie token
  4. Subsequent requests check this cookie via isAuthed()
  5. Protected routes use Auth middleware

Monitor Routes

RouteMethodAuthDescription
/GETNoLogin page or admin dashboard (if authenticated)
/api/loginPOSTNoAuthenticate with username/password
/api/logoutGETYesLogout and invalidate session
/admin/statusGETYesServer status endpoint (CPU, memory, sessions, etc.)
/messagesGETYesServer logs endpoint
/docsGETNoServes documentation
/imgsGETNoServes images
example 
// Start server with monitor
app.listen(true, () => {
  console.log('Main server: http://localhost:3000');
  console.log('Monitor: http://localhost:3001');
});

Create a .env file with:

env 
ADMIN_USERNAME=admin

Admin Sessions

The monitor tracks admin sessions with the following structure:

typescript 
type AdminSessions = {
  id: string;          // Unique session ID
  ip: string;          // Client IP address
  userAgent: string;   // Browser user agent
  creation: number;    // Timestamp of session creation
  isValid: boolean;    // Session validity flag
  request: number;     // Request counter
}

Complete Example

typescript 
import Server from 'tinny-backend';

const app = new Server({ 
  port: 3000, 
  hostname: '0.0.0.0',
  ServerName: 'MyAPI'
});

// Decorators
app.decorate('startTime', Date.now());
app.decorate('config', { version: '1.0.0' });

// Middleware
const auth = (req, res) => {
  const session = res.getCookie('session_id');
  if (!session) {
    res.send(401, { error: 'Unauthorized' });
  }
};

// Routes
app.add({
  method: 'GET',
  path: '/',
  handler: (req, res) => {
    const startTime = app.useDecorator('startTime');
    res.send(200, { 
      status: 'running',
      uptime: Date.now() - startTime.data
    });
  }
});

app.add({
  method: 'GET',
  path: '/api/users/:id',
  handler: (req, res) => {
    const userId = req.params.id;
    res.send(200, { userId, query: req.queries });
  },
  middelWares: [auth]
});

app.add({
  method: 'POST',
  path: '/api/data',
  handler: (req, res) => {
    res.addCookie('data_processed', 'true');
    res.send(201, { 
      received: req.body,
      cookies: res.getAllCookies()
    });
  }
});

// Static files
await app.servDir('./public', 'static');

// Start server with monitor
app.listen(true, () => {
  console.log('Server: http://localhost:3000');
  console.log('Monitor: http://localhost:3001');
});
